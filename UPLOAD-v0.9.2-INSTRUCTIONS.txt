Cassette Motion Pro v0.9.2 - Installer Workflow Hardening

This update makes GitHub Actions more reliable for future installer builds.

What changed:

- Keeps v0.9.1 measurement tool label improvements.
- Keeps Guided Capture.
- The Windows bundle now uploads even if installer creation fails.
- The Windows bundle always includes:

  INSTALLER-BUILD-STATUS.txt

- If installer creation succeeds, the Windows bundle includes:

  CassetteMotionPro-Portable-0.9.2.zip
  CassetteMotionPro-Installer-0.9.2.exe
  INSTALLER-BUILD-STATUS.txt

- If installer creation fails, the Windows bundle still uploads with:

  CassetteMotionPro-Portable-0.9.2.zip
  INSTALLER-BUILD-STATUS.txt

- The status file tells you whether installer creation succeeded or failed.
- The NSIS installer version is now passed from the GitHub workflow version, so
  future updates are less likely to mismatch version numbers.
- Updates app/build/installer/report versions to 0.9.2.

Upload steps:

1. Open your GitHub repository:
   https://github.com/spooled/cassette-motion-pro

2. Upload everything from this package to the repo root.

   Important:
   - Include the hidden .github folder.
   - The workflow file must end up here:

     .github/workflows/build.yml

3. Commit with this message:

   Add v0.9.2 installer workflow hardening

4. Go to Actions.

5. Open the newest run.

6. Download:

   CassetteMotionPro-Windows-0.9.2

What to check:

1. Open the Windows bundle.
2. Open INSTALLER-BUILD-STATUS.txt.
3. If it says SUCCESS, confirm the installer exe is inside the bundle.
4. If it says FAILED, send the Actions run link or the status text so we can fix
   the exact installer error.
5. Confirm the portable zip is still present either way.
